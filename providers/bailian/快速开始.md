# Bailian 模型状态管理 - 快速开始

## 功能说明

系统会自动跟踪哪些模型失败了（403错误），并将这些信息保存到 `bailian_model_state.json` 文件中。即使重启服务，这些状态也会被保留。

## 查看当前状态

### 方法1：命令行工具（推荐）

```bash
# 查看当前模型状态
uv run python -m providers.bailian
```

输出示例：
```
============================================================
Bailian Model State
============================================================

State File: bailian_model_state.json
Last Updated: 2026-04-28 17:30:00 UTC

Current Working Model:
  ✓ qwen-turbo

Available Models (2):
    1. qwen-turbo
    2. qwen-long

Failed Models (2):
  ✗ 1. qwen-max
  ✗ 2. qwen-plus

============================================================
```

### 方法2：直接查看JSON文件

```bash
cat bailian_model_state.json
```

### 方法3：查看日志

在 `server.log` 中搜索以下关键词：
- `BAILIAN_INIT:` - 启动时的状态
- `BAILIAN_MODEL_SUCCESS:` - 成功使用的模型
- `BAILIAN_STATE:` - 状态变更日志

## 管理状态

### 重置失败的模型

如果你想重新尝试之前失败的模型（比如额度已刷新）：

```bash
uv run python -m providers.bailian --reset
```

### 清除所有状态

完全重置，删除状态文件：

```bash
uv run python -m providers.bailian --clear
```

### 手动编辑

直接编辑 `bailian_model_state.json` 文件，然后重启服务。

## 工作流程示例

假设配置了：
```env
BAILIAN_MODELS="qwen-max,qwen-plus,qwen-turbo,qwen-long"
```

### 第一次请求
- 尝试 `qwen-max` → 403错误
- 自动标记为失败
- 切换到下一个模型

### 第二次请求
- 尝试 `qwen-plus` → 403错误
- 自动标记为失败
- 切换到下一个模型

### 第三次请求
- 尝试 `qwen-turbo` → 成功！
- 记录为当前工作模型
- 状态文件更新

### 重启服务后
- 系统加载状态文件
- 自动跳过 `qwen-max` 和 `qwen-plus`
- 只尝试 `qwen-turbo` 和 `qwen-long`
- **自动更新 `.env` 文件**：
  ```env
  MODEL="bailian/qwen-turbo"
  BAILIAN_MODELS="bailian/qwen-turbo,bailian/qwen-long"
  ```

## 优势

✅ **持久化**：重启服务后状态仍然保留  
✅ **自动化**：无需手动干预，系统自动跟踪  
✅ **透明**：随时可以查看当前状态  
✅ **灵活**：可以随时重置或清除状态  
✅ **高效**：不会重复尝试已知失败的模型  
✅ **自动同步**：服务启动时自动更新 `.env` 文件中的配置  

## 常见问题

**Q: 状态文件在哪里？**  
A: 在项目根目录下的 `bailian_model_state.json`

**Q: 这个文件会被提交到Git吗？**  
A: 不会，已经在 `.gitignore` 中排除了

**Q: 如何知道当前用的是哪个模型？**  
A: 运行 `uv run python -m providers.bailian` 查看

**Q: 额度刷新后怎么办？**  
A: 运行 `uv run python -m providers.bailian --reset` 重置状态

**Q: 可以手动指定使用哪个模型吗？**  
A: 可以编辑 `bailian_model_state.json`，设置 `current_working_model` 字段
